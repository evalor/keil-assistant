## 使用说明

### 添加知识
当用户请求**add knowledge**时，我会：
1. 分析要保存的知识类型和规模
2. 如果是大量知识，创建分类目录
3. 创建知识文件：`K001-knowledgename.md`（碎片）或 `category/K001-knowledgename.md`（分类）
4. 更新此索引文件
5. 对于大型知识，可能需要多个会话分步完成

### 知识编号规则

- **碎片化知识**: `K0001`, `K0002`, `K0003`, ...（全局连续）
- **分类知识**: 每个分类目录内独立编号 `K001`, `K002`, `K003`, ...

### 知识分类

知识库支持按主题分类，例如：
- `VS Code扩展开发/` - VS Code API相关知识
- `Keil工具链/` - Keil特定知识
- `TypeScript模式/` - 代码模式和最佳实践
- `性能优化/` - 性能相关技巧
- 等等...

### 搜索知识

使用 `#tool:<grep_search>` 在知识库内搜索相关内容。
